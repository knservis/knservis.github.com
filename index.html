Computing as profession

Currently I am employed as a Senior Software Developer at InnoGames GmbH in Hamburg, Germany until June 2014 when I will be returning to Australia. I have had the opportunity to learn a lot of new technologies and expand on previous knowledge on this role. Most importantly I had the opportunity to use Hadoop, Hive, Storm, Java and Scala. The company allows employees some time for personal development and own projects, and I used that time, among other things to take Martin Odersky's coursera courses "Founamentals of Functional Programming with Scala" and "Reactive Programming with Scala".

Previously I was employed as a Software Developer at HIE, Brisbane. My duties involved maintaining and improving a number of web applications replated to the Australian national eHealth record that the company is deploying. The deployment platform is Sinatra, the OS is Linux, the main programming language used is Ruby and it is all provisioned using Chef. In my job I try to use the best practices that I know of such as Test and Behaviour Driven Development without becoming religious about them. I try to avoid reinventing the wheel and try to dissuade others form doing so, as I think that in general one is best off using code that others have written, tested and are using, when possible. If there are issues with an OS library I am using I try to submit a patch.

My main interests are web information systems, machine learning and artificial intelligence. Most of my professional experience is in web development and most of the courses I have studied had something to do with machine learning and related areas such as data mining and AI in general. Apart from uni courses I have found Andrew Ng's "Machine Learning" coursera course exceptional and Peter Norvig's and Sebastian Thrun's "AI" coursera course really useful. Some pages or web applications that I have helped create or have created are: http://www.mirrabooka.com, http://www.zentro.com.au, https://myapnic.net (users only).
